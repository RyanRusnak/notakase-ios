# Notakase

A typography-first, keyboard-driven markdown notes app — and a publisher — built
natively for **macOS and iOS** in SwiftUI. Recreated from the
[Claude Design](https://claude.ai/design) handoff (`Notakase.dc.html` +
`Notakase iOS.dc.html`).

## Structure

Two app targets share one local Swift package:

```
Notakase.xcodeproj          # generated by XcodeGen from project.yml
project.yml                 # the source of truth for the project
Packages/NotakaseCore/      # shared package: model, themes, markdown, store
  Sources/NotakaseCore/
    Note.swift              # note model + word count
    SeedData.swift          # seed notes, folders, glyphs
    Theme.swift             # 4 Omarchy palettes + SwiftUI colors
    Markdown.swift          # block + inline markdown parser
    MarkdownView.swift      # shared SwiftUI rendering (blocks, inline, source)
    Store.swift             # NotakaseStore (ObservableObject) + deep links
    Shortcuts.swift         # keyboard-sheet content
  Tests/NotakaseCoreTests/  # parser unit tests
Sources/NotakaseMac/        # macOS app — the two-pane desktop layout
Sources/NotakaseiOS/        # iOS app — the Library + Reader flow
```

The core package holds everything platform-agnostic (data, theming, the markdown
parser, and the shared SwiftUI block/inline renderer). Each app owns only its
platform's chrome:

- **macOS** — the two-pane desktop: sidebar tree, breadcrumb + shortcut hints,
  three views (READ / WRITE / PUBLISH), a status bar with a theme picker, a `⌘K`
  command palette, a `?` shortcut sheet, and modal vim-style keys handled through
  a window-local key monitor.
- **iOS** — a touch-first Library of grouped note cards with search and a floating
  `+`, pushing into a Reader with rendered markdown, an inline editor (`✎`), and a
  theme menu (`Aa`).

## Prerequisites

- **Xcode 16+** (developed against Xcode 26, iOS 26 / macOS 26 SDKs)
- **XcodeGen** — `brew install xcodegen`

## Build & run

```bash
xcodegen generate            # (re)generate Notakase.xcodeproj from project.yml
open Notakase.xcodeproj       # then pick the NotakaseMac or NotakaseiOS scheme
```

Or from the command line:

```bash
# macOS
xcodebuild -scheme NotakaseMac -destination 'platform=macOS' build

# iOS simulator
xcodebuild -scheme NotakaseiOS \
  -destination 'platform=iOS Simulator,name=iPhone 15 Pro' build

# core package tests
cd Packages/NotakaseCore && swift test
```

> Whenever you add or remove source files, re-run `xcodegen generate`. The
> `.xcodeproj` is git-ignored and always regenerated from `project.yml`.

## Themes

Nine palettes swap live via SwiftUI: **Tokyo Night** (default), **Catppuccin**,
**Gruvbox**, **Everforest**, plus **Ubuntu**, **Osaka Jade**, **Catppuccin Latte**
(a light theme), **Pulsar**, and **Archwave** — the latter five ported from the
[todarchy-ios](https://github.com/RyanRusnak/todarchy-ios) todo app so both apps
share a look. On macOS press `t` to cycle or use the status-bar picker; on iOS use
the theme pill / `Aa` menu. Light vs. dark drives `preferredColorScheme`
automatically.

## Sync folder

Point Notakase at a sync folder (macOS: **Settings ⌘,** or the status-bar
folder button; iOS: the **gear** in the Library header). The choice is persisted
as a security-scoped bookmark and restored on launch.

Each note is stored as its own **Automerge** document — `note_<id>.automerge` —
so the CLI, the Linux client and this app all read and write the same CRDT files
and merge cleanly through Dropbox / iCloud Drive / Syncthing. The `.automerge`
bytes are the source of truth; there is no parallel markdown on disk.

Document schema (ROOT map): `body` (Text — the markdown), `path` (String, e.g.
`Journal/2026-07-13.md`), `created` / `modified` (Int, ms epoch), `deleted`
(Boolean tombstone), `version` (Int).

- **Path → note:** the `path` field maps onto the tree at full depth —
  `A/B/C/D/note.md` nests four folders deep; a root-level file lands in "Notes".
  Title is the body's first `# heading` (else the filename). The sidebar renders
  this hierarchy recursively, so nesting is unbounded.
- **Writes are load-modify-save:** editing a note splices its `body` Text and
  bumps `modified`, preserving the document's history and stored `path`; a new
  note gets a fresh 16-byte base64url id and a full schema.
- **`deleted: true` notes are hidden** but their tombstone is kept so deletes
  propagate to other devices.
- **Auto-import:** a folder watcher re-reads the vault whenever the sync daemon
  drops in / removes note files, so notes written on another device appear
  automatically. **Sync now** (⌘K palette) forces a re-read.
- **Empty folder?** It's seeded once with the sample notes so there's something to
  look at. Clearing the folder reverts to the in-memory seed.

The engine lives in `AutomergeVault` (core), watched by `FolderWatcher`, and is
covered by unit tests. `MarkdownVault` remains for its path/title helpers.

## Embedding todokase tasks

A note can embed a live task list from [todokase](https://github.com/RyanRusnak/todarchy-ios)
with a fenced `todokase` block:

````markdown
```todokase
project: notakase
status: open
```
````

`status` is `open` (default), `done`, or `all`; a bare line is shorthand for the
project name. The block renders as a live checklist read from todokase's
`tasks.automerge` — point at it once under **Settings → Todokase Tasks**. It's
read-only (notakase never writes the task store). The reader is `TodokaseTasks`
(core) and the renderer is `TodokaseBlockView`.

## Letting Claude create notes (MCP)

`notakase-mcp/` is a local [MCP](https://modelcontextprotocol.io) server (stdio
JSON-RPC, mirroring `todarchy-mcp`) that lets Claude Desktop / Claude Code
create and read notes directly. It reuses `AutomergeVault`, so notes it writes
are byte-identical to the apps' and sync through the same folder.

Tools: `add_note` (title / folder / body), `list_notes`, `get_note`.

Build, install, and register:

```bash
cd notakase-mcp && swift build -c release
cp .build/release/notakase-mcp ~/bin/notakase-mcp
```

Then add to `~/Library/Application Support/Claude/claude_desktop_config.json`:

```json
"notakase": {
  "command": "/Users/<you>/bin/notakase-mcp",
  "env": { "NOTAKASE_FOLDER": "/path/to/your/notakase_sync" }
}
```

`NOTAKASE_FOLDER` points at the sync folder; `NOTAKASE_MCP_READ_ONLY=1` disables
writes. Restart Claude Desktop to pick it up.

## Deep links

The iOS app registers the `notakase://` URL scheme. `notakase://wiki/<title>`
opens that note — this is also what makes `[[wikilinks]]` tappable across both
platforms.

```bash
xcrun simctl openurl booted "notakase://wiki/Omarchy%20Setup"
```

  (git-ignored) as the source-of-truth reference.
